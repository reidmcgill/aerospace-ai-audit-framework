# Contributing

This project is in early-stage doctoral research. Contributions, feedback, and discussion are welcome.

## How to Contribute

1. **Open a Discussion** — For questions, ideas, or feedback on the research direction, use [GitHub Discussions](https://github.com/reidmcgill/aerospace-ai-audit-framework/discussions).

2. **File an Issue** — For specific technical suggestions, bugs, or feature requests.

3. **Submit a Pull Request** — For code, documentation, or schema contributions.

## Areas Where Input is Especially Valuable

- Decision logging schema design and standards alignment
- Mapping AI audit requirements to existing certification frameworks (DO-178C, ARP4754A)
- Digital twin architecture patterns for regulatory compliance
- Real-world experience with AI certification in aerospace, automotive, or medical domains

## Code Standards

- Python 3.11+
- Linting via `ruff`
- Type hints encouraged
- Docstrings on all public functions
